# Evaluation

## 功能评估以及自迭代
当前项目目录下的 evaluation 文件夹包含了一系列可运行的评估脚本，每个脚本为对应功能模块的测试用例。评估和自迭代流程需要执行以下几部分操作，保障软件代码实现达到预期状态：
1. 执行 evaluation 目录下的 sh 脚本
2. 判断脚本执行情况，如果脚本执行过程中发现错误则尝试修复对应代码
3. 获取执行脚本后，功能模块输出的相关性能指标数据以及日志记录，判断是否存在优化空间
4. 如果分析出功能模块可进一步优化，在保障功能正确性的情况下修改代码以提升性能
5. 如果修改了代码，重复步骤 1～4 的迭代流程；如果无需修改代码则停止循环
