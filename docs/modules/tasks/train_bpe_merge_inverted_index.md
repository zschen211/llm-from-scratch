# Merge acceleration based on inverted indices

## 背景和目标
当前 merge 阶段的迭代统计在每轮迭代时都会遍历 chunk 文件进行统计，导致每次 merge 迭代都将大量系统资源和计算开销花费在遍历训练数据上。我希望落地一个优化方案，在 merge 第一次迭代时构建倒排索引，并从第二次迭代开始不再遍历全量训练数据，而是基于倒排索引判断下一轮的 merge pair。请帮我用 Rust 代码完成这一方案的代码实现，并根据下方要求完成测试评估

## 实现路径
1. 仅在 merge 第一次迭代时遍历全量训练数据，第一次迭代结束后完成倒排索引的构建
2. 从第二次迭代开始，基于倒排索引统计 pair count 并完成剩余的 merge 操作
3. 每轮统计均需要找到当前频次最高的 pair 并进行合并操作，这一过程可以通过大顶堆数据结构完成以实现 O(logN) 的查找效率

## 评估标准
1. 从第二次迭代开始，应该完全停止遍历 chunk 文件，并观察到明显的训练加速
2. 保障大顶堆和倒排索引在迭代过程中的正确性，合理判断数据结构中是否存在过期数据
3. 如果内存无法塞下所有辅助加速的数据结构，优先剔除无关的内存对象占用（比如第一次遍历完chunk文件后，应该移除chunk以节省内存空间），如果无法省出内存空间则 offload 到磁盘
